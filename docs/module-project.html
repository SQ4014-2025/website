Getting Started

Creating an entire project may seem daunting. Here are some questions that you should think about as you start:

  • What will your software do? What features will it have? How will it be executed?
  • What new skills will you need to acquire? What topics will you need to research?
  • If working with one or two classmates, who will do what?
  • In the world of software, most everything takes longer to implement than you expect. And so itâ€™s not uncommon to accomplish less in a fixed amount of time than you hope. What might you consider to be a good outcome for your project? A better outcome? The best outcome?

Consider making goal milestones to keep you on track.

How to Submit

Outline: The final project offers students the opportunity to apply programming skills in developing their own software solution. This project encourages creativity and innovation, allowing students to choose any programming language and build an application that is personally meaningful, solves real-world problems, or potentially impacts the community or world at large. While students can opt to work individually, collaboration with up to two classmates is permitted, with the expectation that each member contributes equally to the projectâ€™s design and implementation. The complexity of the project should reflect the size of the group, with group projects expected to be more intricate than individual ones. The final project is assessed based on the problemâ€™s complexity and scope, the effectiveness of the implemented solution, creativity, and the overall quality of the code. For group projects, the assessment also considers individual contributions and the collaborative effort. Individuals may choose to complete the project by themselves or to group with up to two other people (i.e., â€œgroupâ€ size is 1, 2, or 3 students).

For each project, a short 3-minute video that demonstrates the project in action (either using slides, screenshots, voiceovers and/or live action) should be created. This should be uploaded to YouTube and the URL shoul be shared in the project README.md. Videos can be uploaded as an â€œunlistedâ€ video.

Projects should be hosted via a GitHub Repository. In addition to all project associated files, this should include a README.md file that includes the following: * Project Title * Names of group members (or individual) * GitHub usernames of all group members * Link to video demo * Link to GitHub Repo * Project description * To do list/task list (For groups this should also include information on which group member contributed to the task)

Only one GitHub repository and only one video needs to be created per group.

Deliverables: A word document or PDF containing the README.md information should be submitted to the turn it in link on the VLE module page. Each individual needs to upload this separately.
